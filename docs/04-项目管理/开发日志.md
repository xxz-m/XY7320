### 已完成事项

#### 1. Bootloader PB8 按键消抖

在 `XY_bootloadr/Core/Src/main.c` 中新增 PB8 按键检测参数：

```c
#define BOOT_KEY_GPIO_PORT       GPIOB
#define BOOT_KEY_GPIO_PIN        GPIO_PIN_8
#define BOOT_KEY_PRESSED_LEVEL   GPIO_PIN_RESET
#define BOOT_KEY_WAIT_TIMEOUT_MS 10000U
#define BOOT_KEY_SCAN_PERIOD_MS  10U
#define BOOT_KEY_DEBOUNCE_MS     50U
#define BOOT_KEY_LOG_PERIOD_MS   1000U
```

PB8 当前为 GPIO 上拉输入，低电平表示按下。Bootloader 启动后等待约 10 秒，期间每 10 ms 扫描一次，连续稳定低电平 50 ms 后认为按键有效。

#### 2. Bootloader 启动决策

当前启动判断流程：

```text
Bootloader 启动
  ↓
初始化 GPIO / DMA / USART1 / USART2
  ↓
USART1 输出调试日志
  ↓
等待 PB8 按键 10 秒
  ↓
按键有效：停留在 Bootloader，开启 USART2 DMA + IDLE 接收
  ↓
按键无效：检查 APP 是否有效
  ↓
APP 有效：跳转 APP
  ↓
APP 无效：停留在 Bootloader，开启 USART2 DMA + IDLE 接收
```

当前只预留"按键进入升级模式"的基础框架，后续 APP 写升级标志、上位机日期版本号、升级完成同步版本号等逻辑还未接入。

#### 3. 调试串口日志

启动和等待期间会输出：

```text
bootloader start
debug uart ready
upgrade key: PB8, pressed level: LOW
hold PB8 to enter upgrade mode, wait 10000 ms...
boot wait start, timeout = 10000 ms
boot wait: xxxx ms left, PB8=x
PB8 low detected, debounce...
PB8 debounce ok
boot wait timeout
APP valid, jump to APP
APP invalid, stay bootloader
waiting upgrade...
```


```c
#define BOOTLOADER_SIZE     (64 * 1024)
#define APP_PART_SIZE       (ONCHIP_FLASH_SIZE - BOOTLOADER_SIZE)
```

## 遇到并解决的问题

### user.h 缺失

`common.h` 包含 `user.h`，工程里没有，于是补了最小配置。

### RTOS 头文件误包含

因为未定义宏在 `#if` 中会被当成 0，导致误进入 RTOS 分支。

修复方式：

```c
#if defined(RTOS_USING_RTTHREAD) && (USING_RTOS_TYPE == RTOS_USING_RTTHREAD)
```

### GCC 不支持原始 update_flag 放置写法

补了 GCC 分支：

```c
volatile uint64_t update_flag __attribute__((section(".noinit"), aligned(8)));
```

### bootloader_port.c 重复定义

曾同时存在两个 `bootloader_port.c`，导致链接重复定义。

最终保留：

```text
MOTA/Core/bootloader_port.c
```

删除重复文件：

```text
MOTA/Port/bootloader_port.c
```
