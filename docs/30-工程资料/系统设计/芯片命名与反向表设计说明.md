# 芯片命名与反向表设计说明

> 目的：统一回答"为什么 GSM 选 ADL5906A、DMR 正向选 ADL8361、DMR 反向选 ADL8317"、"反向通道为什么是 `1600 - raw` 转换"以及"芯片型号在代码与文档中如何保持命名一致"。
> 适用读者：阅读 `Domain/measurement/`、`Domain/model/calibration_table.cpp` 之前需要建立芯片选型与命名映射关系的同事。

## 1. 选型结论

| 模式 | 芯片 | 选型理由 |
|------|------|----------|
| GSM | ADL5906A | RSSI 输出，dBm 标定，单电源 5V，覆盖 GSM 全功率段 |
| DMR 正向（413/457 MHz） | ADL8361 | 50 MHz ~ 6 GHz 对数检波器，覆盖 DMR UHF 频段，正向功率测量 |
| DMR 反向（413/457 MHz） | ADL8317 | 50 MHz ~ 6 GHz 对数检波器，与 ADL8361 同系列，专用于反向功率 |

三颗芯片都属于 ADI 的对数检波器家族，输出都是 mV 级别的直流电压，硬件接口一致，软件查表路径也基本一致，可以共用一份 `Calibration_AdcToDbm` 入口。

## 2. 三芯片手册要点（只列与代码相关的）

> 说明：以下要点只列与代码查表/转换逻辑相关的部分，不复制 ADI 数据手册内容，避免版权问题。完整规格请参考 ADI 官方数据手册。

### 2.1 ADL5906A（GSM）

- 频段：50 MHz ~ 6 GHz
- 输出：RSSI 电压，与输入功率呈对数关系
- 标定：dBm 输出，斜率 ~ 18 mV/dB
- 代码对应：
  - 表名：`ad_5906_mv_adc` / `ad_5906_dbmX10`
  - 函数：`Calibration_5906AdcToDbm`
  - 测量类：`GSMPowerMeasurement`

### 2.2 ADL8361（DMR 正向）

- 频段：LF ~ 6 GHz
- 输出：与输入功率呈对数关系的电压
- 应用：发射功率检测 / 接收信号强度
- 代码对应：
  - 表名：`ad_adl8361_mv_adc` / `ad_adl8361_dbmX10`
  - 函数：`Calibration_ADL8361AdcToDbm`
  - 测量类：`DMRPowerMeasurement`（P1/P3 通道）

### 2.3 ADL8317（DMR 反向）

- 频段：LF ~ 6 GHz
- 输出：与输入功率呈对数关系的电压
- 应用：与 ADL8361 配对，用于反向功率检测
- 代码对应：
  - 表名：`ad_adl8317_mv_adc` / `ad_adl8317_dbmX10`
  - 函数：`Calibration_ADL8317AdcToDbm`
  - 测量类：`DMRPowerMeasurement`（P2/P4 通道，需要先做 `1600 - raw` 转换）

## 3. 反向通道 1600 - raw 转换的来源

### 3.1 设计动机

反向功率检测的物理特性：

- 反射功率越大 → 入射到检波器的射频能量越大 → 检波器输出电压越大 → ADC 读数越大。
- 反射功率越小（接近 0 dBm 反射）→ 入射能量越小 → 输出电压越小 → ADC 读数越小。

但是从 PCB 走线和运放偏置上看，反向通道的运放配置是 **反相 + 同向偏移**，即"原始功率越大 → 输出电压越低、ADC 读数越小"。这与正向通道完全相反。

为了在软件层让两套通道共用 `Calibration_AdcToDbm` 入口（同样的"power ↑ → value ↑"单调关系），反向通道在 measurement 内部对原始值做 `1600 - raw` 翻转，让翻转后的值与正向通道一样具有"power ↑ → value ↑"的单调性。

### 3.2 为什么是 1600

`1600` 这个常量来自 ADC 满量程 3.3V / 12-bit ADC = 0 ~ 4095 范围内的中点近似（实际运放输出摆幅在 0 ~ 3300 mV，系数 1600 是按运放设计中心点取的）。具体值需要在 PCB 量产前按运放实际偏置微调。

### 3.3 当前代码实现位置

```cpp
/* Domain/measurement/dmr_power_measurement.cpp */
const uint16_t raw_p2v = p2v;
const uint16_t raw_p4v = p4v;
const uint16_t lookup_p2v = (p2v <= 1600u) ? (uint16_t)(1600u - p2v) : 0u;
const uint16_t lookup_p4v = (p4v <= 1600u) ? (uint16_t)(1600u - p4v) : 0u;
```

- 当原始 ADC ≤ 1600（合理范围）：翻转后正常查表。
- 当原始 ADC > 1600（运放饱和或异常）：按 0 查表，结果是 dBm 最低端，触发 fall-through。
- 这个分支只在 DMR 测量内部存在，对协议层与 AdcService 上层透明。

### 3.4 待办

- 系数 1600 是否需要按芯片手册/PCB 实际偏置微调 → 参见 `校准与测量链路总览.md` 的"已知待办 TBD-4"。

## 4. 命名规范

> 这是项目内必须遵守的命名约定，目的是让"芯片型号 → 代码实体 → 文档条目"保持一一对应。

| 层级 | 命名模板 | 示例 |
|------|----------|------|
| 校准表（const） | `ad_<chip>_mv_adc` / `ad_<chip>_dbmX10` | `ad_adl8361_mv_adc` / `ad_adl8361_dbmX10` |
| 校准表长度宏 | `<CHIP>_TABLE_LENGTH` | `ADL8361_TABLE_LENGTH` |
| 校准查表函数 | `Calibration_<Chip>AdcToDbm` / `Calibration_<Chip>AdcToUw` | `Calibration_ADL8361AdcToDbm` |
| 校准表宏开关 | `CALIB_ENABLE_<MODE>_TABLE` | `CALIB_ENABLE_DMR_TABLE` |
| 测量类 | `<Mode>PowerMeasurement` | `DMRPowerMeasurement` / `GSMPowerMeasurement` |
| 测量数据结构 | `<Mode>PowerData_t` | `DMRPowerData_t` / `GsmPowerData_t` |
| 链路补偿 | `<MODE>_<CH>_OFFSET_DB` | `DMR_P1_OFFSET_DB` |

> 注意：`<chip>` 字段对 ADL8361/ADL8317 保留小写（`adl8361` / `adl8317`），对 ADL5906A 也保留小写（`5906`），但宏与函数中的大写部分遵循 STM32 编码规范的 `ALL_CAPS` 习惯（`ADL8361_TABLE_LENGTH` 而不是 `Adl8361_Table_Length`）。

## 6. 历史命名回顾（避免回退）

> 这部分保留是为了防止有人按历史命名（`ad_8361_*` / `Calibration_8361AdcToDbm`）重新引入。

| 旧名称（2026-07-07 之前） | 新名称（2026-07-08 起） | 备注 |
|---------------------------|--------------------------|------|
| `ad_8361_mv_adc` / `ad_8361_dbmX10` | `ad_adl8361_mv_adc` / `ad_adl8361_dbmX10` | 加上 `adl` 前缀，避免与 AD8361（ADI 另一颗对数放大器）混淆 |
| `ad_8361_reverse_mv_adc` / `ad_8361_reverse_dbmX10` | `ad_adl8317_mv_adc` / `ad_adl8317_dbmX10` | 反向通道真实芯片是 ADL8317，不是 ADL8361 反向模式 |
| `Calibration_8361AdcToDbm` | `Calibration_ADL8361AdcToDbm` | 同上 |
| `Calibration_8361ReverseAdcToDbm` | `Calibration_ADL8317AdcToDbm` | 同上 |
| `AD_8361_TABLE_LENGTH` | `ADL8361_TABLE_LENGTH` | 宏名大写 + 加上 `L` |
| `AD_8361_REVERSE_TABLE_LENGTH` | `ADL8317_TABLE_LENGTH` | 同上 |

如果未来发现旧符号被错误地留在代码或文档中，按"以代码为准"原则修复，文档层面不会回滚到旧名称。
