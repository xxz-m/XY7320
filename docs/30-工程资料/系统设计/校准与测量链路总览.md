# 校准与测量链路总览

> 目的：从 BSP/adc 一直走到 protocol_service，完整描述 XY7320 当前的"ADC 采样 → 滤波 → measurement → 查表 → 协议上传"端到端数据流。
> 适用读者：阅读 ADC/measurement/校准表/协议上传相关源码之前需要先建立全局概念的同事。

## 1. 链路总图

```text
┌────────────────┐
│  CubeMX DMA    │  TIM/DMA 双缓冲半传输完成/全传输完成中断
│  BSP/adc       │  BspAdc_* 只搬运到 bsp_adc.c 的双缓冲
└──────┬─────────┘
       │ bsp_adc_read_snapshot()
       ▼
┌────────────────────────┐
│  Services/adc_service  │  AdcService 单例
│  AdcService::StartTaskA│  按 Mode 选择 Oscilloscope::ScopeMode_t
│  AdcService::ProcessTaskA  维护 m_snapshot[6][ADC_SIZE_MAX]
└──────┬─────────────────┘
       │ m_snapshot → Oscilloscope::waveLimitFilter()
       ▼
┌────────────────────────┐
│  Domain/filter/        │  Oscilloscope 是纯算法 + 滤波
│  Oscilloscope          │  输出 m_pep[6]、m_wave[6] 等
└──────┬─────────────────┘
       │ 每个 tick 把滤波后的 PEP 喂给 measurement
       ▼
┌─────────────────────────────────────────────────────────┐
│  Domain/measurement/                                    │
│  ┌────────────────────────┐  ┌────────────────────────┐ │
│  │  DMRPowerMeasurement   │  │  GSMPowerMeasurement   │ │
│  │  .Update(p1v, p2v,    │  │  .Update(p5, p6,      │ │
│  │  p3v, p4v, now, out)  │  │  now, out)             │ │
│  │  内置 1600-p2v/p4v 转换│  │  仅两通道             │ │
│  └───────────┬────────────┘  └───────────┬────────────┘ │
└──────────────┼───────────────────────────┼──────────────┘
               │ out: DMRPowerData_t       │ out: GsmPowerData_t
               ▼                           ▼
┌──────────────────────────────────────────────────────────┐
│  Domain/model/calibration_table                          │
│  Calibration_ADL8361AdcToDbm / Calibration_ADL8317AdcToDbm│
│  Calibration_5906AdcToDbm   ← 由 measurement 调用       │
│  Calibration_AdcToDbm / Calibration_AdcToUw  统一入口     │
└──────┬───────────────────────────────────────────────────┘
       │ dBm*10 / uW 落进 out->w1x_uw_413 / dbm1x100_413
       ▼
┌────────────────────────┐
│  AdcService::m_xxxData │  Services 层只持有最新结果
│  AdcService::GetXxx()  │  对外只读 API
└──────┬─────────────────┘
       │ protocol_service 拉取并打包
       ▼
┌────────────────────────┐
│  Services/uart_service │
│  ProtocolService       │  按 0xF0/0xA1/... 命令打包
│  + Domain/protocol     │  protocol_xy::EncodePacket
└────────────────────────┘
```

## 2. 芯片与校准表对应关系

| 模式 | 芯片 | 表名（const uint16_t / int16_t） | 函数 | 测量类 | 输出单位 |
|------|------|-----------------------------------|------|--------|----------|
| GSM | ADL5906A | `ad_5906_mv_adc` / `ad_5906_dbmX10` | `Calibration_5906AdcToDbm` | `GSMPowerMeasurement` | dBm × 10 / uW |
| DMR 正向 | ADL8361 | `ad_adl8361_mv_adc` / `ad_adl8361_dbmX10` | `Calibration_ADL8361AdcToDbm` | `DMRPowerMeasurement` | dBm × 10 / uW |
| DMR 反向 | ADL8317 | `ad_adl8317_mv_adc` / `ad_adl8317_dbmX10` | `Calibration_ADL8317AdcToDbm` | `DMRPowerMeasurement` | dBm × 10 / uW |
| DMR 反向 转换 | — | `1600 - raw` | 在 `DMRPowerMeasurement::Update` 内部 | DMR 反向专属 | ADC / mV |

校准表宏开关位于 `Common/config/calibration_config.h`：

- `CALIB_ENABLE_GSM_TABLE`：默认 `1`，关闭后 `ad_5906_*` 与 `Calibration_5906AdcToDbm` 不编译。
- `CALIB_ENABLE_DMR_TABLE`：默认 `1`，关闭后 `ad_adl8361_*` / `ad_adl8317_*` 与两个 `Calibration_*AdcToDbm` 不编译。

## 3. 路径一：GSM 测量（AdcTaskB）

```text
HAL_ADC_ConvCplt
  └─> BspAdc_DmaXferCpltCb  (BSP/adc/bsp_adc.c)
        └─> BspAdc_MarkReady(channel)
              └─> AdcService::ProcessTaskB
                    ├─> BspAdc_PeekSnapshot(p5, p6)
                    ├─> Oscilloscope::waveLimitFilter(ScopeMode_GSM, p5, p6, &m_wave, &m_pep)
                    └─> m_gsmPowerMeasurement.Update(m_pep[4], m_pep[5], HAL_GetTick(), &m_gsmPowerData)
                          └─> Calibration_5906AdcToDbm(adc, GSM_OFFSET_DB, &dbm)
                          └─> Calibration_DbmToUw(dbm, &uw)
                          └─> 短时掉信号 hold + dbm1x100 / uw1x 维护
                          └─> PEP 峰值维护
```

GSM 走 `task_state_adc_b` 业务模式，由 ModeManager 在收到 `Mode::GSM` 时切入；离开 GSM 模式时 `m_gsmPowerMeasurement.Reset()` 复位 hold / PEP。

## 4. 路径二：DMR 测量（AdcTaskA）

```text
HAL_ADC_ConvCplt
  └─> BspAdc_DmaXferCpltCb
        └─> AdcService::ProcessTaskA
              ├─> BspAdc_PeekSnapshot(p1, p2, p3, p4)
              ├─> Oscilloscope::waveLimitFilter(ScopeMode_DMR, p1..p4, &m_wave, &m_pep)
              └─> m_dmrPowerMeasurement.Update(
                      m_pep[0], m_pep[1], m_pep[2], m_pep[3],
                      HAL_GetTick(), &m_dmrPowerData)
                    ├─> if (p2v <= 1600) lookup_p2v = 1600 - p2v  (反向 ADL8317)
                    ├─> if (p4v <= 1600) lookup_p4v = 1600 - p4v
                    ├─> Calibration_ADL8361AdcToDbm(p1v, P1_OFFSET_DB, &dbm1)
                    ├─> Calibration_ADL8317AdcToDbm(lookup_p2v, P2_OFFSET_DB, &dbm2)
                    ├─> Calibration_ADL8361AdcToDbm(p3v, P3_OFFSET_DB, &dbm3)
                    ├─> Calibration_ADL8317AdcToDbm(lookup_p4v, P4_OFFSET_DB, &dbm4)
                    ├─> Calibration_DbmToUw(dbmX, &uwX)
                    ├─> 短时掉信号 hold（lastValidTs + 1500ms）
                    ├─> 1.5% 抖动保持
                    └─> PEP 峰值维护（dmrPowerData->pepUw_*）
```

DMR 走 `task_state_adc_a` 业务模式（与 GSM 共用 BSP/adc，但 Mode 区分 Oscilloscope 滤波参数与 measurement 入口）。当前 DMR 走的是 ProcessTaskA 扩展；若后续 GSM 测量本身有进一步需求，可考虑独立为 AdcTaskC。

## 5. 单位约定

| 量 | 类型 | 含义 | 出现位置 |
|------|------|------|----------|
| 校准表 ADC 端 | `uint16_t` | 静态查表坐标（mV） | `ad_*_mv_adc[]` |
| 校准表 dBm 端 | `int16_t` | 定点 `dBm * 10` | `ad_*_dbmX10[]` |
| measurement 中间值 | `float` | dBm / uW | `DMRPowerMeasurement::Update` 内部 |
| 上传 dbm | `int32_t` | `round(dBm * 100)`，即 `dBm * 100` | `dmrPowerData->dbm1x100_413` 等 |
| 上传 uW | `uint32_t` | 微瓦 | `dmrPowerData->w1x_uw_413` 等 |
| 链路补偿 | `float` | dBm 偏置，按通道单独可调 | `DMR_P1_OFFSET_DB = 30.72f` 等 |

注：校准表使用 `dBm * 10` 是为了在 `int16_t` 范围（−327.0 dBm ~ +327.0 dBm）内保持 0.1 dBm 精度；上传协议使用 `dBm * 100` 进一步保留 0.01 dBm 精度。

## 6. 校准表宏开关

```c
/* Common/config/calibration_config.h */
#define CALIB_ENABLE_GSM_TABLE    1U
#define CALIB_ENABLE_DMR_TABLE    1U
```

- `CALIB_ENABLE_GSM_TABLE = 0` → `ad_5906_*` 与 `Calibration_5906AdcToDbm` 不参与编译。
- `CALIB_ENABLE_DMR_TABLE = 0` → `ad_adl8361_*` / `ad_adl8317_*` 与两个 `Calibration_*AdcToDbm` 不参与编译。
- 同时关闭两个宏时，可以节省 ~4 KB Flash（取决于表大小）。

注意：当前 GSM 与 DMR 测量类都假设对应宏已开启；如果关闭其中之一，对应 measurement 的 Update 路径需要同步收敛（属于"待办"段，不在本轮文档修正范围）。

## 7. PEP 与短时保持

`GSMPowerMeasurement` 与 `DMRPowerMeasurement` 共用一套 PEP / 保持策略：

- **瞬时归零 hold**：当一帧的 dBm 突然掉到 < 阈值（如 -60 dBm），不立即把 out 清零，而是把 `out` 保持为上一帧有效值，更新 `lastValidTs`。
- **1500ms 超时**：从 `lastValidTs` 起算 1500ms 内没有新的有效信号 → out 清零。
- **1.5% 抖动保持**：当前帧与上一帧差值 < 1.5% → 上一帧保持；否则更新为新值（抑制小幅抖动，避免读数跳动）。
- **PEP 峰值**：每次 Update 取新帧的 `dBm` 与历史 PEP 比较，留下较大者；PEP 仅在模式切换或显式 Reset 时清零。

PEP 清零策略目前由 mode_manager 在切模式时调用 `measurement.Reset()` 完成；手动清零 API 暂未实现（属于"待办"段）。

## 8. 链路补偿 offset_db

`measurement.Update` 接收 `offset_db` 参数作为链路补偿，含义是把芯片自身标定误差 + PCB 走线误差 + 射频前端误差一起补偿掉。当前默认值：

```cpp
constexpr float GSM_OFFSET_DB   = 35.10f;  // 来自 ADL5906A 标定 + PCB 补偿
constexpr float DMR_P1_OFFSET_DB = 30.72f;  // DMR 413 MHz 正向
constexpr float DMR_P2_OFFSET_DB = -2.20f;  // DMR 413 MHz 反向
constexpr float DMR_P3_OFFSET_DB = 30.55f;  // DMR 457 MHz 正向
constexpr float DMR_P4_OFFSET_DB = -2.42f;  // DMR 457 MHz 反向
```

这 4 个 DMR 补偿值是"分通道独立维护"的，不收敛成一个 `DMR_OFFSET_DB`，因为 4 个通道的 PCB 走线长度差异较大，共享一个偏置会引入较大误差。

补偿值精调需要外接标准信号源对比，参见 [[20-开发记录/踩坑记录]]（待补）。
