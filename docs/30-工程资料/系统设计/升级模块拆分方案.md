# 升级模块拆分方案

> [!info] 文档目的
> 将当前集中在 `app_update_config.c` 中的升级逻辑，按 [[10-项目规范/架构规范|架构规范]] 拆分为 BSP / Domain / Services / App 四层，使升级链路达到与 LED 链路同等的分层清晰度。

## 1 当前问题

### 1.1 现状

当前升级链路集中在三个文件：

```text
Core/Src/app_update_config.c    — 接收 + 轮询 + 解析 + 写配置 + ACK + 复位
Core/Src/app_version_config.c   — 协议解析 + Flash 读写 + 槽位管理
App/usecase/task_update.cpp     — 简单包装 Poll()
```

`app_update_config.c` 混合了三层职责：

| 职责 | 具体内容 | 应属层 |
|------|----------|--------|
| 硬件操作 | DMA 缓冲区管理、IDLE 中断处理、DMA 启停、溢出检测 | BSP |
| 协议/业务 | 轮询接收标志、帧解析、发送 ACK、触发复位 | Services |
| 存储 | 初始化 A1 版本、写 A2 版本 | Services |

### 1.2 对比 LED 链路

LED 链路已经比较干净：

```text
Task_LED → Led_service → BSP_GPIO     ✅ 三层清晰
```

升级链路目前是：

```text
Task_UpdateConfig → app_update_config.c（一坨）   ❌ 全揉在一起
```

## 2 拆分目标

### 2.1 最终调用链

```text
Task_UpdateConfig            App/usecase/          只调 service.Update()
      │
      ▼
Update_service               Services/             封装升级流程：轮询、协调、ACK、复位
      │
      ├──→ Version_frame     Domain/               版本帧协议解析（纯计算，无硬件依赖）
      │
      ├──→ Version_store     Services/             A1/A2 版本配置的 Flash 读写
      │      └──→ Bsp_flash  BSP/                  Flash 擦/读/写
      │
      └──→ Bsp_uart_rcv      BSP/                  DMA 收发、IDLE 中断、缓冲区
```

### 2.2 依赖方向

```text
App/usecase          → Services/update_service     ✅ (App → Services)
Services/update      → Domain/version_frame        ✅ (Services → Domain)
Services/update      → Services/version_store      ✅ (Services → Services，同层允许)
Services/update      → BSP/uart                    ✅ (Services → BSP)
Services/version     → BSP/flash                   ✅ (Services → BSP)
Domain/version_frame → (无依赖)                     ✅ (Domain 不依赖任何层)
BSP/uart             → HAL                         ✅ (BSP → HAL)
BSP/flash            → HAL                         ✅ (BSP → HAL)
```

> [!success] 零反向依赖

## 4 各模块详细设计

### 4.1 BSP/flash — Flash 硬件操作

#### 来源

从 `app_version_config.c` 搬出的纯硬件操作：

```c
HAL_FLASH_Unlock()
HAL_FLASH_Lock()
HAL_FLASH_Program(FLASH_TYPEPROGRAM_WORD, addr, word)
HAL_FLASHEx_Erase(&erase_init, &sector_error)
```

#### 接口

```c
// BSP/flash/bsp_flash.h

int BspFlash_Read(uint32_t addr, uint8_t *buf, uint32_t len);
int BspFlash_Write(uint32_t addr, const uint8_t *buf, uint32_t len);
int BspFlash_EraseSector(uint8_t sector);
```

#### 实现要点

- `BspFlash_Read`：内部 Flash 直接 `memcpy`
- `BspFlash_Write`：按字编程，逐 4 字节写入并回读校验
- `BspFlash_EraseSector`：封装 `HAL_FLASHEx_Erase`

> [!note] BSP 不知道的事
> 版本配置在哪、A1/A2 是什么、擦写顺序策略。

---

### 4.2 BSP/uart — 串口 DMA 接收

#### 来源

从 `app_update_config.c` 搬出的 DMA + IDLE 中断处理：

```c
static uint8_t app_update_rx_buf[64];
static uint8_t app_update_proc_buf[64];
static volatile uint16_t app_update_rx_len = 0;
static volatile uint8_t  app_update_rx_done = 0;
static volatile uint8_t  app_update_rx_overflow = 0;

App_UpdateConfig_StartReceive()         // 开 DMA + 使能 IDLE
App_UpdateConfig_HandleUartIdleIrq()    // IDLE 中断全流程
```

#### 接口

```c
// BSP/uart/bsp_uart_rcv.h

void     Bsp_UartRcv_Init(UART_HandleTypeDef *huart, uint8_t *buf, uint16_t size);
void     Bsp_UartRcv_Start(void);
bool     Bsp_UartRcv_IsFrameReady(void);
uint16_t Bsp_UartRcv_GetFrameLength(void);
void     Bsp_UartRcv_CopyFrame(uint8_t *dst);
void     Bsp_UartRcv_ClearFlag(void);
void     Bsp_UartRcv_SendAck(const uint8_t *data, uint16_t len);
void     Bsp_UartRcv_HandleIdleIrq(UART_HandleTypeDef *huart);
```

#### 实现要点

- 接收缓冲区由调用方（`update_service`）分配，通过 `Init` 传入
- 内部维护一份 `s_proc_buf` 用于中断中拷贝帧数据
- `s_frame_ready` 标志由中断置位，由 `ClearFlag()` 清除
- `HandleIdleIrq` 在 `stm32f4xx_it.c` 的 `USART2_IRQHandler` 中调用

> [!note] BSP 不知道的事
> 帧里是什么协议、版本帧长什么样、收到后该不该复位。

---

### 4.3 Domain/version_frame — 版本帧协议解析

#### 来源

从 `app_version_config.c` 搬出的纯协议解析：

```c
static int AppVersionConfig_ParseAsciiVersion(const uint8_t *data, uint64_t *version)
static int AppVersionConfig_ParseFrame(const uint8_t *data, size_t len, ...)
```

#### 接口

```c
// Domain/version_frame/version_frame.h

#define VERSION_FRAME_FLAG_NEED_DOWNLOAD  0x00U
#define VERSION_FRAME_FLAG_DOWNLOADED     0x01U

typedef struct {
    uint64_t version;
    uint8_t  flag;
} VersionFrame;

int VersionFrame_Parse(const uint8_t *data, size_t len, VersionFrame *out);
```

#### 协议格式

```text
XYVH (4字节) + yyyyMMddHHmm (12字节ASCII) + flag (1字节) + XYVT (4字节) = 21 字节
```

#### 实现要点

- 纯函数：字节流 → `VersionFrame` 结构体
- 不依赖任何 HAL、Flash、UART
- 拿到任何平台都能编译

> [!success] Domain 层特性
> 这个文件不依赖任何硬件，是纯领域知识。

---

### 4.4 Services/version_store — 版本配置存储

#### 来源

从 `app_version_config.c` 搬出的 Flash 存储逻辑：

```c
#define APP_VERSION_CONFIG_ADDR       0x080E0000U
#define APP_VERSION_SLOT_A1_OFFSET    0x00000000U
#define APP_VERSION_SLOT_A2_OFFSET    0x00000040U

static int AppVersionConfig_WriteWord(...)
static int AppVersionConfig_WriteSlot(...)
static int AppVersionConfig_Save(...)
static void AppVersionConfig_MakeSlot(...)
AppVersionConfig_Read(...)
AppVersionConfig_IsSlotValid(...)
AppVersionConfig_UpdateA1(...)
AppVersionConfig_UpdateA2(...)
AppVersionConfig_ShouldEnterUpgrade(...)
```

#### 接口

```c
// Services/version_store/version_store.h

typedef struct {
    uint32_t magic;
    uint64_t version;
    uint8_t  flag;
    uint8_t  reserved[7];
    uint32_t tail;
} VersionSlot;

typedef struct {
    VersionSlot a1;
    VersionSlot a2;
} VersionConfig;

int VersionStore_Read(VersionConfig *config);
int VersionStore_WriteA1(uint64_t version, uint8_t flag);
int VersionStore_WriteA2(uint64_t version, uint8_t flag);
int VersionStore_ShouldEnterUpgrade(void);
int VersionStore_IsSlotValid(const VersionSlot *slot);
```

#### 实现要点

- 知道版本配置存在 `0x080E0000`（Sector 11）
- 知道 A1 在偏移 `0x00`，A2 在偏移 `0x40`
- 知道擦写策略：先擦整个扇区，再写回 A1 + A2
- **不直接调 `HAL_FLASH_*`**，而是通过 `BspFlash_*` 操作

> [!note] Services 与 BSP 的边界
> Services 知道"存什么、怎么组织"，BSP 知道"怎么操作 Flash"。
> 就像 LED 链路：`BSP_GPIO` 知道怎么翻转引脚，但不知道什么是"闪烁模式"。

### 4.5 Services/update_service — 升级流程编排

#### 来源

从 `app_update_config.c` 搬出的业务逻辑：

```c
App_UpdateConfig_Init()         // 初始化 A1
App_UpdateConfig_Poll()         // 轮询 + 处理
App_UpdateConfig_ProcessFrame() // 解析 + 写 A2 + ACK + 复位
```

#### 接口

```c
// Services/update_service/update_service.h

void Update_service_Init(void);
void Update_service_Update(void);
```

#### 实现要点

`Init()` 负责：

1. 调用 `Bsp_UartRcv_Init()` 初始化串口接收
2. 调用 `Bsp_UartRcv_Start()` 启动 DMA 接收
3. 调用 `VersionStore_WriteA1()` 写当前运行版本

`Update()` 负责：

1. 检查 `Bsp_UartRcv_IsFrameReady()`
2. 取帧：`GetFrameLength()` + `CopyFrame()` + `ClearFlag()`
3. 解析：`VersionFrame_Parse()`（Domain 层）
4. 存储：`VersionStore_WriteA2()`（Services 层）
5. 决策：如果 `flag == NEED_DOWNLOAD`，发 ACK 并复位

> [!tip] Services 是"胶水层"
> 调 Domain 解析协议，调 Services 存储版本，调 BSP 收发数据。自己不碰任何硬件细节。

---

### 4.6 App/usecase/task_update — 任务入口

#### 改动

从：

```c
extern "C" void Task_UpdateConfig(void *arg)
{
    App_UpdateConfig_Poll();
    OS_DelayMs(10);
}
```

改为：

```c
extern "C" void Task_UpdateConfig(void *arg)
{
    Update_service_Update();
    OS_DelayMs(10);
}
```

> [!success] 与 task_led.cpp 完全同风格
> 一行调用 + 一行延时。

## 6 拆完后完整调用链

```text
main.c
  ├── MX_USART2_UART_Init()          (CubeMX 生成)
  ├── App_Main_Init()
  │     ├── Led_service::Init()
  │     └── Update_service_Init()
  │           ├── Bsp_UartRcv_Init(&huart2, buf, 64)     ← BSP
  │           ├── Bsp_UartRcv_Start()                     ← BSP
  │           └── VersionStore_WriteA1(当前版本)            ← Services → BSP/flash
  │
  └── App_Main_Start()
        ├── OS_CreateTask(Task_LED, ...)
        └── OS_CreateTask(Task_UpdateConfig, ...)

Task_UpdateConfig (每 10ms)
  └── Update_service_Update()
        ├── Bsp_UartRcv_IsFrameReady()     ← BSP
        ├── Bsp_UartRcv_GetFrameLength()   ← BSP
        ├── Bsp_UartRcv_CopyFrame(frame)   ← BSP
        ├── Bsp_UartRcv_ClearFlag()        ← BSP
        ├── VersionFrame_Parse(frame, ...)  ← Domain（纯计算）
        ├── VersionStore_WriteA2(...)       ← Services → BSP/flash
        ├── Bsp_UartRcv_SendAck("XYA1")    ← BSP
        └── NVIC_SystemReset()

USART2_IRQHandler
  └── Bsp_UartRcv_HandleIdleIrq()          ← BSP
```

## 7 建议动手顺序

| 步骤 | 做什么 | 风险 |
|------|--------|------|
| 1 | 新建 `BSP/flash/bsp_flash.c/.h`，把 Flash 操作搬过去 | 低，纯搬 |
| 2 | 新建 `BSP/uart/bsp_uart_rcv.c/.h`，把 DMA 接收搬过去 | 低，纯搬 |
| 3 | 新建 `Domain/version_frame/version_frame.c/.h`，把协议解析搬过去 | 低，纯搬 |
| 4 | 新建 `Services/version_store/version_store.cpp/.h`，把存储逻辑搬过去 | 中，需要改调 BspFlash |
| 5 | 新建 `Services/update_service/update_service.cpp/.h`，把流程逻辑搬过去 | 中，需要改调三个新模块 |
| 6 | 改 `task_update.cpp` 调 `Update_service` | 一行改动 |
| 7 | 改 `App_Main.cpp` 接入 `Init` | 两行改动 |
| 8 | 改 `main.c` 去掉直接调升级函数 | 删几行 |
| 9 | 改 `stm32f4xx_it.c` 的 IRQ 指向 `Bsp_UartRcv` | 改一行调用 |
| 10 | 改 `CMakeLists.txt` 新增 include 目录 | 加几行 |
| 11 | 删除旧文件 `app_update_config.*` 和 `app_version_config.*` | 确认无引用后删除 |
| 12 | 编译验证 | — |

> [!tip] 每步独立编译
> 每一步都是独立的，做完一步编译一次，不会一次改太多导致找不到问题。
