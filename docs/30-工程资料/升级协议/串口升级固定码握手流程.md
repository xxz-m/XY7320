# 串口升级固定码握手流程

本文记录当前已经跑通的 `APP -> Bootloader -> XY7320Host` 升级流程：APP 前置阶段使用主协议 `0xF0` 升级握手，BootLoader 阶段继续使用 `XYB1/XYB2/XYB3` 固定码。

## 升级链路总览

参与端：

- `XY7320 APP`：通过 USART2 接收主协议升级握手，写入升级版本配置后复位
- `XY_bootloadr`：读取升级配置，停留升级模式，接收头包和 APP bin 数据，写入 `0x08010000` 后跳转 APP
- `XY7320Host`：发送主协议升级握手、头包、固件数据，按固定码推进自动升级状态机

## 固定码定义

- `主协议 ACK`：APP 已接收升级握手命令，即将写配置并复位
- `XYB1`：Bootloader 已进入升级模式，USART2 已准备接收
- `XYB2`：Bootloader 已接收头包并擦除 APP 区，正在等待 APP 数据
- `XYB3`：Bootloader 已接收完 APP，校验有效，即将跳转 APP

## 自动升级时序

1. Host 发送主协议升级握手：`cmd=0xF0, payload=yyyyMMddHHmm + flag`
2. APP 返回主协议 ACK
3. Bootloader 启动并返回 `XYB1`
4. Host 发送 12 字节头包
5. Bootloader 擦除 APP 区后返回 `XYB2`
6. Host 连续发送 APP bin
7. Bootloader 校验有效后返回 `XYB3`
8. Host 提示升级完成，Bootloader 跳转 APP

## APP 升级握手格式

- `origin_port`：`0x01`（PC）
- `goal_port`：`0x22`（XY7320）
- `model`：`0x02`（unWrite）
- `cmd`：`0xF0`
- `payload[0..11]`：12 字节 ASCII 版本号 `yyyyMMddHHmm`
- `payload[12]`：1 字节 `flag`

APP 入口位于：`XY7320/Services/uart_service/protocol_service.cpp` 与 `XY7320/Services/upgrade/update_service.cpp`

## 头包格式

上位机发送给 Bootloader 的头包为 12 字节小端序：

- `magic`：4 字节 `0x41505055`
- `app_size`：4 字节 APP bin 文件大小
- `crc32`：4 字节 APP bin CRC32

Bootloader 处理入口位于：`XY_bootloadr/MOTA/Core/simple_update.c`

## 上位机自动状态机

位于：

- `tools/XY7320Host/backend/FirmwareUploader.cpp`
- `tools/XY7320Host/backend/FirmwareUploader.h`

状态包括：

- `WaitAppVersionAck`
- `WaitBootReady`
- `WaitHeaderAck`
- `SendingFirmware`
- `WaitFinishAck`

## 结论

当前已实机跑通：APP 收主协议升级握手 -> APP 复位 -> Bootloader 停留升级 -> 头包握手 -> 固件写入 -> Bootloader 跳转 APP -> APP 正常运行。
