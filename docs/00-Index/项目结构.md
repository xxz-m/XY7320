### Core

```text
Core/Inc
Core/Src
Core/Startup
```

`Core` 保留 CubeMX 生成代码和工程主入口。当前 `main.c` 的职责已经收敛为：

- 初始化 GPIO / DMA / USART / TIM
- 初始化协作式调度器 `OS_Init()`
- 初始化升级接收 `App_UpdateConfig_Init()` / `App_UpdateConfig_StartReceive()`
- 初始化 App 层 `App_Main.Init()`
- 创建任务并启动调度器 `App_Man_Start()` / `OS_Start()`

### App

```text
App/
├─ App_Main.cpp
├─ App_Main.h
├─ command/                  # .gitkeep 占位，预留业务命令入口
├─ mode_manager/
│  ├─ fsm.h              # 轻量级状态机框架（State/Event 基类）
│  ├─ mode_types.h       # 模式枚举、事件类型、具体事件类
│  ├─ mode_manager.h     # ModeManager 单例接口
│  └─ mode_manager.cpp   # ModeManager 实现（已完整落地）
└─ usecase/
   ├─ task_led.cpp / task_led.h              # LED 任务入口
   ├─ task_update.cpp / task_update.h        # 升级轮询任务入口
   ├─ task_state_idle.h / task_state_idle.cpp
   ├─ task_state_dmr.h / task_state_dmr.cpp
   ├─ task_state_gsm.h / task_state_gsm.cpp
   └─ task_gnss.h / task_gnss.cpp
```

`App` 层负责业务编排和任务注册。

- `App_Main.cpp/h`：统一初始化服务并创建任务
- `task_led.cpp`：周期驱动 LED 服务
- `task_update.cpp`：周期驱动串口升级轮询逻辑
- `mode_manager/`：ModeManager + TinyFSM 状态机，管理 Idle/DMR/GSM/GNSS 模式切换
- `task_state_dmr.cpp`：DMR 状态，等待 PA15/TIM2_CH1 周期脉宽有效后采集正反功率
- `task_state_gsm.cpp`：GSM 状态，等待 PA2/TIM5_CH3 周期脉宽有效后采集正反功率
- `task_gnss.cpp`：GNSS 状态，驱动 GNSS Service 的 USART3 DMA+IDLE 接收
- `task_state_*.cpp`：各模式的具体业务实现（entry/exit/tick）
- `command/`：当前为占位目录，预留业务命令入口

ModeManager 状态机架构：

```text
ModeManager (单例)
  ├── Init()      → 默认进入 Idle
  ├── Tick()      → 每 1ms 由 OS 任务驱动，调用当前状态的 tick()
  └── RequestSwitch(event) → 统一切换入口：exit旧状态 → 切换 → entry新状态

状态：Idle ↔ DMR ↔ GSM ↔ GNSS（由 ModeManager 统一切换）
```

详见 [[30-工程资料/系统设计/XY7320-ModeManager与TinyFSM设计说明]] 与 [[30-工程资料/系统设计/校准与测量链路总览]]。

```text
BSP/
├─ adc/                          # 新增
│  ├─ bsp_adc.c

│  └─ bsp_gpio.h

│  └─ bsp_tim_os.h
└─ uart/                         # 新增
   ├─ bsp_uart.c
   ├─ bsp_uart.h
   ├─ bsp_uart_rcv.c
   └─ bsp_uart_rcv.h
```

`BSP` 层负责板级硬件适配。

- `bsp_gpio.*`：封装 GPIO 开关、翻转、读状态
- `bsp_tim_os.*`：把 TIM14 配置为 OS Tick，并在中断中驱动 `OS_Tick()`
- `bsp_adc.*`：**新增**，只做 DMA 搬运 + 双缓冲 ready/Resume 流控，不感知 TaskA/TaskB 业务
- `bsp_uart.*` + `bsp_uart_rcv.*`：**新增**，封装 USART 收发 + 接收环形缓冲
- `bsp_flash.*`：内部 Flash 擦写（升级 / 配置存储）
- `dma/`：当前为占位目录，DMA 配置在 CubeMX 生成的 `Core/` 中

### Services

```text
Services/
├─ adc_service/
│  ├─ adc_service.cpp
│  └─ adc_service.h
├─ led_service/
│  ├─ led_service.cpp

   ├─ update_service.h
   ├─ version_store.cpp
   └─ version_store.h
```

`Services` 层负责把底层能力组织成应用可直接使用的服务。

- `adc_service/`：**新增**，AdcService 单例；维护 `Mode` 与 `Oscilloscope::ScopeMode_t` 对应关系；持有本地 `m_snapshot[6][ADC_SIZE_MAX]` 避免 Oscilloscope 原地修改 BSP 缓冲；提供 `GetDmrPowerData()` / `GetGsmPowerData()` 给协议层拉取
- `led_service/`：LED 模式机（`OFF / ON / BLINK / BREATHE`），内部调用 `BSP_GPIO_*`
- `log/`：日志服务，基于 Middleware/nanoprintf 实现
- `uart_service/protocol_service.*`：主协议解析与分发（升级握手 `0xF0` + 业务命令字）
- `upgrade/update_service.*` + `version_store.*`：升级握手 + 版本槽（A1/A2）读写

### Domain

```text
Domain/
├─ filter/
│  ├─ Oscilloscope.cpp
│  └─ Oscilloscope.h
├─ measurement/
│  ├─ dmr_power_measurement.cpp
│  ├─ dmr_power_measurement.h
│  ├─ gsm_power_measurement.cpp
│  └─ gsm_power_measurement.h
├─ model/
│  ├─ calibration_table.cpp
│  └─ calibration_table.h
└─ protocol/
   ├─ configurations.h
   ├─ protocol_xy.cpp
   └─ protocol_xy.h
```

`Domain` 层放算法、模型、协议纯处理，禁止依赖 `BSP`、`Middleware`、HAL。

- `filter/Oscilloscope.*`：只负责 waveLimitFilter 滤波算法，不感知业务
- `measurement/dmr_power_measurement.*` / `gsm_power_measurement.*`：ADC/mV → dBm/uW 换算 + 短时掉信号保持 + PEP 峰值；不感知协议上传
- `model/calibration_table.*`：校准表 + 插值换算（`Calibration_AdcToDbm` / `Calibration_AdcToUw`），不感知模式
- `protocol/protocol_xy.*`：主协议编解码

详见 [[30-工程资料/系统设计/校准与测量链路总览]] 与 [[30-工程资料/系统设计/芯片命名与反向表设计说明]]。

### Middleware

```text
Middleware/
└─ nanoprintf/
   └─ nanoprintf.h
```

当前唯一中间件是 nanoprintf（轻量级 printf 替代）。`Middleware` 层放第三方库或通用协议栈，尽量少改原库；后续如需引入 mbedTLS / cJSON / FreeRTOS 等放这里。

### System

```text
System/
├─ event/                  # 占位
├─ queue/                  # 占位
└─ scheduler/
   ├─ os.c
   └─ os.h
```

`System` 层做调度、事件、队列、Tick 等基础设施，不承载具体业务。

- `scheduler/os.c/h`：**已落地**的轻量级协作式调度器
  - 非抢占式
  - 任务函数每次只执行一步逻辑
  - 通过 `OS_DelayMs()` 让出 CPU
  - 不直接依赖 HAL
- `event/`、`queue/`：当前为占位目录，后续按业务需要落地

## 当前工程状态

当前仓库主线状态如下：

```text
1. APP / Bootloader / 上位机 升级握手链路已跑通（主协议 0xF0 握手）
2. APP 已从 main.c 轮询式结构过渡到 App + Services + BSP + System / Domain / Middleware / Common 分层
3. LED 模块已完成较干净的任务化拆分
4. 升级握手已接入 ProtocolService，更新路径使用主协议命令分发
5. APP 与 Bootloader 已开始分别维护各自的版本配置
6. GSM 功率测量链路第一版闭环：AdcService → Oscilloscope → GSMPowerMeasurement → CalibrationTable
7. DMR 功率测量链路第一版闭环：4 通道 → DMRPowerMeasurement → CalibrationTable（ADL8361 正向 + ADL8317 反向）
8. 校准表统一使用 dBm*10 定点格式，由 CALIB_ENABLE_GSM_TABLE / CALIB_ENABLE_DMR_TABLE 独立控制
9. 芯片命名规范化：GSM = ADL5906A；DMR 正向 = ADL8361；DMR 反向 = ADL8317
```
