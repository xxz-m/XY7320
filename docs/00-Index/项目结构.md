# 项目结构说明

## 总目录

```text
E:\XY7320\XY7320
```

当前仓库由 Bootloader、APP、上位机工具和项目文档四部分组成。

```text
XY_bootloadr/                 Bootloader 工程
XY7320/                       APP 工程
tools/XY7320Host/             Qt 上位机串口升级工具
docs/                         项目文档
```

## XY_bootloadr

```text
XY_bootloadr/
├─ Core/
├─ Drivers/
├─ MOTA/
└─ Components/
```

这是当前 STM32F407 Bootloader 工程。

- `Core/`：CubeMX 生成代码、启动文件、Bootloader 入口
- `Drivers/`：STM32 HAL 与 CMSIS
- `MOTA/`：升级核心逻辑、协议处理、Flash Port 适配
- `Components/`：日志与轻量格式化组件

当前 Bootloader 已不再依赖仓库根目录的 `Common/app_version_config.*`，版本配置逻辑已独立放入：

```text
XY_bootloadr/Core/Inc/app_version_config.h
XY_bootloadr/Core/Src/app_version_config.c
```

## XY7320

```text
XY7320/
├─ App/
├─ BSP/
├─ Common/
├─ Core/
├─ Domain/
├─ Drivers/
├─ Middleware/
├─ Services/
└─ System/
```

这是当前 APP 工程，链接地址为 `0x08010000`。

### Core

```text
Core/Inc
Core/Src
Core/Startup
```

`Core` 保留 CubeMX 生成代码和工程主入口。当前 `main.c` 的职责已经收敛为：

- 初始化 GPIO / DMA / USART / TIM
- 初始化协作式调度器 `OS_Init()`
- 初始化升级接收 `App_UpdateConfig_Init()` / `App_UpdateConfig_StartReceive()`
- 初始化 App 层 `App_Main.Init()`
- 创建任务并启动调度器 `App_Man_Start()` / `OS_Start()`

### App

```text
App/
├─ App_Main.cpp
├─ App_Main.h
├─ command/
├─ mode_manager/
└─ usecase/
   ├─ task_led.cpp
   ├─ task_led.h
   ├─ task_update.cpp
   └─ task_update.h
```

`App` 层负责业务编排和任务注册。

- `App_Main.cpp`：统一初始化服务并创建任务
- `task_led.cpp`：周期驱动 LED 服务
- `task_update.cpp`：周期驱动串口升级轮询逻辑

当前升级任务已经接入调度器，但 `App_UpdateConfig_Poll()` 内部仍属于过渡态，后续还可以继续拆到 `update_service` / `update_protocol` 一类模块。

### BSP

```text
BSP/
├─ gpio/
│  ├─ bsp_gpio.c
│  └─ bsp_gpio.h
└─ tim/
   ├─ bsp_tim_os.c
   └─ bsp_tim_os.h
```

`BSP` 层负责板级硬件适配。

- `bsp_gpio.*`：封装 GPIO 开关、翻转、读状态
- `bsp_tim_os.*`：把 TIM14 配置为 OS Tick，并在中断中驱动 `OS_Tick()`

### Services

```text
Services/
├─ adc_service/
├─ led_service/
│  ├─ led_service.cpp
│  └─ led_service.h
├─ report/
└─ uart_service/
```

`Services` 层负责把底层能力组织成应用可直接使用的服务。

当前已经成型的是 `led_service`：

- 维护 LED 模式：`OFF / ON / BLINK / BREATHE`
- 内部调用 `BSP_GPIO_*` 完成硬件动作
- 提供 `Init()`、`Set_mode()`、`Update()` 等接口给 App 层调用

### System

```text
System/
├─ event/
├─ queue/
└─ scheduler/
   ├─ os.c
   └─ os.h
```

`System` 层当前实现的是一个轻量级协作式调度器。

特点：

- 非抢占式
- 任务函数每次只执行一步逻辑
- 通过 `OS_DelayMs()` 让出 CPU
- 不直接依赖 HAL

### Common / Domain / Middleware

这三层目前主要处于目录和边界预留阶段。

- `Common/`：放公共配置、类型、工具
- `Domain/`：后续放测量模型、算法、规则
- `Middleware/`：后续放第三方协议栈或通用组件

## 串口升级链路在 APP 中的落点

当前 APP 侧串口升级相关代码主要分布在：

```text
XY7320/Core/Src/app_update_config.c
XY7320/Core/Src/stm32f4xx_it.c
XY7320/App/usecase/task_update.cpp
XY7320/Core/Src/app_version_config.c
```

职责划分如下：

- `app_update_config.c`：升级接收状态、DMA+IDLE 接收、版本帧处理、ACK 和复位
- `stm32f4xx_it.c`：`USART2_IRQHandler()` 中转到 `App_UpdateConfig_HandleUartIdleIrq()`
- `task_update.cpp`：把 `App_UpdateConfig_Poll()` 接入协作式调度器
- `app_version_config.c`：版本槽读写、帧解析、升级标志判断
