# 当前进度

> [!success]
> 这里不记录所有历史，只记录当前主线进度和最近几个关键变化。

## 当前主线状态

- 固定码握手升级链路已跑通
- APP 新分层框架已落地第一版
- Bootloader 与 APP 的版本配置模块已拆开
- 文档结构已完成重构，按用途 + 主题分类
- 配置集中管理已落地（bsp_config.h / app_config.h）
- 日志服务已移植（nanoprintf + LogService）
- **ModeManager + TinyFSM 状态机已落地，模式切换功能验证通过**
- **上位机串口调试工具已添加模式切换按钮，协议模式值统一来自 configurations.h**
- **UART2 DMA deinit 问题已修复，升级握手复位后 Bootloader 可正常启动**
- **ADC BSP/Service 已接入 DMR/GSM 状态，先完成周期脉宽有效判断，再采集正反功率**
- **上位机串口端口下拉框刷新/选择问题已修复**
- **ADC 平均值计算异常和高频日志导致疑似卡死问题已修复/缓解**
- **调试串口关闭后进入 STM32 System Bootloader 的硬件问题已定位并通过断开自动下载控制电阻修复**
- **GSM 校准表、功率 measurement 和 TaskB 独立入口已完成第一版闭环**
- **串口助手接收区已支持跨多条复制、带时间戳复制、关键字过滤/高亮和自定义竖向胶囊滚动条**
- **DMR 模式第一版闭环已落地（Domain/measurement/dmr_power_measurement + 校准表 ADL8361/ADL8317）**
- **DMR 反向通道 1600-raw 转换已落地（见 dmr_power_measurement.cpp）**
- **校准表统一使用 dBm*10 定点格式，CALIB_ENABLE_GSM_TABLE / CALIB_ENABLE_DMR_TABLE 独立开关落地**
- **芯片命名规范化：GSM = ADL5906A；DMR 正向 = ADL8361；DMR 反向 = ADL8317**
- **TIM2_CH1/PA15 与 TIM5_CH3/PA2 输入捕获已接入 DMR/GSM，支持动态边沿、周期脉宽、有效性和超时判断**
- **DMR/GSM 状态详解已建立，覆盖 OS 任务、FSM 状态、协议任务和采集流程**
- **UART2 ReceiveToIdle DMA + UartTxService 第一版已通过 APP 构建（0x30/0x31/0x32 完整协议帧、固定字段大端序列化、Mode ping-pong、控制 FIFO、非阻塞升级 ACK 排空）；连续收发与错误恢复时序仍待实机验证**

## 近期待推进

- ADC 长时间运行稳定性继续实机验证；若仍卡死，用 ST-Link 读取 Fault 寄存器定位
- ADC 日志输出后续改为非阻塞队列或 DMA 发送，避免调试输出影响实时采集
- 业务主线（采集 -> 上传 -> 显示）待启动
- 旧项目模式迁移（POWER / MOHMS / CALI / GSM）
- 升级模块状态机优化（避免 HAL_Delay 阻塞）
- **实机验证 UartTxService 连续收发、ACK FIFO 保序、Mode 切换丢弃、RX/TX 错误恢复与升级 ACK 排空复位**
- **当前 DMR/GSM 仍上传周期/脉宽占位值，后续替换为 DMRSpec/GSMSpec 真值字段；保持现有显式大端序列化**
- **DMR 测量结果接入协议上传字段（0x? 命令待定，见校准与测量链路总览 TBD-1）**
- **PEP 清零策略需要业务确认：进入 GSM/DMR 时清零、手动清零、还是随日志清空联动（见 TBD-2）**
- **链路补偿值（offset_db）继续精调，对照外接标准信号源比对（见 TBD-3）**
- **反向通道 1600-raw 转换系数是否需要按芯片手册微调（见 TBD-4）**
- **校准表宏开关在产品版本中是否需要默认关闭以节省 Flash（见 TBD-5）**
