## 当前已形成的关键决策

### OS 任务与 FSM 状态边界

- OS 只注册 `Task_LED`、`Task_UpdateConfig` 和 `Task_ModeManager` 三个周期任务。
- `TaskStateDmr`、`TaskStateGsm`、`TaskStateGnss` 和 `TaskStateIdle` 是 FSM 状态，不创建独立 OS 任务。
- 所有模式切换统一通过 `ModeManager::RequestSwitch()`，状态生命周期统一经过 `exit()`、状态指针更新和 `entry()`。
- 详细任务职责记录在 [[30-工程资料/系统设计/XY7320任务详解]]。

### DMR/GSM 测量流程

- DMR 绑定 `PA15/TIM2_CH1`，GSM 绑定 `PA2/TIM5_CH3`。
- 进入 DMR/GSM 状态时先清除对应 A/B 捕获历史，再等待周期和脉宽有效。
- 周期脉宽有效后才启动对应正向/反向功率 ADC；退出状态时停止 ADC。
- 输入捕获中断只采集并排队原始边沿，Service 在任务上下文中完成周期、脉宽、有效性和超时判断。

### 协议命名来源

- 协议模式命令使用 `configurations.h` 中的 `WAIT_MODEL`、`POWER_MODEL`、`POWER_MODEL_GSM` 和 `GPS_MODEL`，禁止在分发逻辑中散落 `0x10` 等硬编码值。
- 代码事件和模式统一使用 `EVT_SWITCH_TO_DMR/GSM`、`MODE_DMR/GSM` 与 `SwitchToDmrEvent/GsmEvent`。
- 删除旧 `TaskStateAdcA/B` 文件和 ADC A/B 状态命名，协议命令字节保持配置文件定义不变。

- APP 不再继续把业务逻辑堆回 `main.c`
- `App_UpdateConfig_Poll()` 已先进入任务框架，再逐步继续拆细
- APP 与 Bootloader 不再长期共享同一个 `app_version_config` 源文件
- 文档结构采用 Obsidian 导航层 + 历史内容保留的方式演进

## 校准与测量链路关键决策（2026-07-08 整理）

- **dBm 统一使用 `dBm * 10` 定点表**：校准表输出端采用 `int16_t` 定点格式，语义为 `dBm * 10`，既保留 0.1 dBm 精度，又避免协议层直接处理 float；上传协议再放大到 `dBm * 100`，保留 0.01 dBm 精度。详见 [[30-工程资料/系统设计/芯片命名与反向表设计说明]]。
- **三芯片方案选型**：GSM 选 ADL5906A（RSSI 输出）；DMR 正向选 ADL8361；DMR 反向选 ADL8317。三颗都属于 ADI 对数检波器家族，硬件接口一致，软件查表可共用入口。详见 [[30-工程资料/系统设计/芯片命名与反向表设计说明]]。
- **校准表统一入口**：`Calibration_AdcToDbm` / `Calibration_AdcToUw` 作为唯一对外接口；`Calibration_Lookup` 按输出单位分发，底层共用同一份 `Calibration_AdcToDbm` 插值逻辑，不重复实现。
- **DMR 反向通道 `1600 - raw` 转换**：反向运放配置是反相 + 同向偏移，原始 ADC 与功率呈反单调。measurement 内部做 `1600 - raw` 翻转后再查 ADL8317 表，让两套通道共用"power ↑ → value ↑"的单调性，协议层与 AdcService 上层对翻转无感知。详见 [[30-工程资料/系统设计/芯片命名与反向表设计说明]]。
- **校准表独立开关**：`CALIB_ENABLE_GSM_TABLE` / `CALIB_ENABLE_DMR_TABLE` 由 `Common/config/calibration_config.h` 集中控制，默认全开；产品版本可单独关闭以节省 Flash。
